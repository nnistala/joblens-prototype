🎯

Core Features
🕷️

Multi-Source Aggregation

Crawls company career pages (Greenhouse, Lever, Workday, Taleo), public job boards, and Google for Jobs structured data. Updated daily with freshness checks.

🔄

Smart Deduplication

3-stage pipeline: exact hash → fuzzy title match (>85% similarity) → semantic cosine similarity. Merges listings from 5 platforms into 1 canonical job card.

🔍

Full-Text Search

OpenSearch-powered search with filters: role, city, remote/hybrid/onsite, experience, salary range, company size, ATS platform, posted date.

📬

Smart Job Alerts

Save a search, get notified when matching jobs appear. Digest frequency: real-time, daily, weekly. Delivered via email; push notifications in Phase 2.

🏢

HR Direct Portal

Companies self-register their career page URL. JobLens crawls it on a schedule, indexes all openings, and drives qualified applicant traffic back.

💰

Apply Flow

Redirect for scraped jobs (opens original source). Direct apply for partnered companies. Candidates always apply at source — JobLens is the discovery layer, not a wall.

Competitive Landscape

How JobLens Wins

  • 🟢Completeness — career pages first means 100% of openings, not a subset
  • 🟢Freshness — verified within 24h; stale jobs flagged and removed
  • 🟢No ghost postings — dedup + freshness kills duplicates and expired listings
  • 🟢India-optimised — filters for INR salary, city tiers, Indian company types
  • 🟢No login wall — browse all jobs without account; sign up only to save/alert

Competitors at a Glance

  • 🔵Naukri — dominates India volume, weak at stealth/leadership roles, login-wall
  • 🔵LinkedIn — best for networking, suffers easy-apply spam & ghost posts
  • 🔵Indeed — aggregator but stale, duplicate issues, weak India focus
  • 🔵Google for Jobs — great SEO, no product experience, no alerts
  • 🔵Instahyre / Cutshort — startup/tech niche, not comprehensive

Scrapy Spider Architecture
🌿

greenhouse.py

Hits boards.greenhouse.io/<company> JSON API. Paginated. Extracts structured job data directly — no HTML parsing needed. Covers 40+ Indian companies.

🎯

lever.py

Hits jobs.lever.co/<company>. Parses JSON embed. Extracts role, team, location, apply URL. Fast — no JS rendering required.

🤖

generic_career_page.py

Playwright-based. Renders JS SPAs. Uses CSS selectors + heuristics to identify job listings. Handles Workday, Taleo, iCIMS, BambooHR, and custom career pages.

🔎

google_jobs_schema.py

Extracts application/ld+json JobPosting schema from company sites. Companies voluntarily publish these — cleanest, most legal data source.

🇮🇳

indeed_india.py

Public listing pages (no login). Respects robots.txt, crawl-delay. Stores metadata + source URL only — not full JD text (copyright).

Crawl Schedule

Celery Beat runs schedules per spider type: career pages every 4h, public boards every 12h, freshness checks every 6h. Configurable via YAML.

Legal Strategy — Tiered Crawling
✅ Tier 1 — Fully Safe
Company career pages (public, voluntary)
Google for Jobs JobPosting schema
HR/company partnerships (opt-in)
Public APIs / RSS/Atom feeds
Greenhouse / Lever public JSON APIs
⚠️ Tier 2 — Proceed with Caution
Public job board pages (no login)
Respect robots.txt + crawl-delay
Store metadata + link only, not full JD
Attribution back to source
hiQ v LinkedIn (2022) precedent
❌ Tier 3 — Avoid
Login-wall content (LinkedIn, Naukri)
Bypassing CAPTCHAs / IP blocks
Full copyrighted JD text storage
Personal/candidate profile data
ToS-violating scraping
Security & Auth
🔑

JWT + OAuth2

Short-lived access tokens (15min) + refresh tokens (7d) stored in httpOnly cookies. OAuth2 via Google and LinkedIn — no password stored for social auth.

🛡️

DPDPA Compliance

India DPDPA 2023: explicit consent for email alerts. No personal candidate data scraped. All user data deletable on request. Data stored in ap-south-1 (India).

🔒

Secrets Management

Zero secrets in code. AWS Secrets Manager for DB passwords, API keys. JWT_SECRET_KEY rotated monthly. ECR image scanning enabled.
